# ADR 017: LR2021 Only — Ban SX1280 from All Source and Documentation

## Status
Accepted (2026-07-22)

## Context
**Recurring problem:** LLMs and AI agents repeatedly write firmware using SX1280 SPI commands and register definitions, even though the project has NEVER used an SX1280 chip. The chip on our boards is the **Semtech LR2021** (LoRa Plus, Gen 4, marketed as "LR1280" in some NiceRF module variants).

This has caused:
- RSSI bugs (SX1280 stores RSSI differently than LR2021)
- Wrong packet status commands (0x024B is SX1280's GET_PACKET_STATUS, LR2021 uses a different register)
- SPI command sequences that don't match the actual hardware
- LLMs wasting time debugging "SX1280 issues" that don't exist
- Confusion across handover sessions — every new LLM session needs re-explanation

**The SX1280 and LR2021 are completely different chips:**

| Aspect | SX1280 (Gen 2) | LR2021 (Gen 4) |
|--------|----------------|-----------------|
| Release | ~2017 | 2025 |
| Frequency | 2.4 GHz only | Sub-GHz + 2.4 GHz + S-Band |
| SPI Command Set | SX128x commands | LR11xx commands (SWDR001 driver) |
| FLRC | Yes (2.4 GHz) | Yes (Sub-GHz + 2.4 GHz) |
| LoRa | 2.4 GHz only | Sub-GHz + 2.4 GHz |
| RSSI Register | GET_PACKET_STATUS (0x024B) | Different command — use RadioLib LR2021 driver |
| Max Sensitivity | -132 dBm | -141.5 dBm |
| Max Data Rate | 200 kbps LoRa / 2 Mbps FLRC | 2.6 Mbps FLRC |

**The SX1280 is NOT a predecessor to the LR2021.** They share the 2.4 GHz band and FLRC modulation name, but the silicon, register maps, and SPI protocols are entirely different.

## Decision

### 1. SX1280 is BANNED from all project source code
- No SX1280 SPI commands (0x024B, 0x0094, etc.)
- No SX1280 register definitions
- No SX1280 constant references
- No `#include` of SX128x headers (except inside RadioLib library itself)

### 2. SX1280 references scrubbed from documentation
- ADR-002 comparison table updated to remove SX1280 as a "reference point"
- `docs/sx1280-repos-reference.md` deleted
- No new documentation may reference SX1280 as if it were our chip

### 3. AGENTS.md updated with CRITICAL WARNING
A prominent warning block at the top of AGENTS.md stating:
- Chip = LR2021 (NOT SX1280)
- Use RadioLib LR2021 driver only (radio.beginFLRC, radio.beginLoRa)
- Never use raw SX1280 SPI commands

### 4. Git pre-commit hook
Add a git hook that blocks any commit containing SX1280 SPI command patterns in source files (0x024B, 0x0094, SX1280_GET, etc.).

## Why LLMs Keep Making This Mistake

1. **Training data bias:** SX1280 is far more common in LoRa/2.4 GHz projects. LLMs default to it.
2. **RadioLib structure:** RadioLib's LR2021 driver inherits from PhysicalLayer, but the SX128x driver family is more prominent in the codebase (.pio/libdeps contains SX128x files as part of RadioLib's module structure).
3. **Existing code contamination:** The 5 raw SPI firmware files (flrc_throughput_*.cpp, flrc_range_rx_*.cpp, flrc_raw_tx_pipe.cpp) were written using SX1280 commands, creating a self-reinforcing loop.
4. **ADR-002 comparison table:** Listing SX1280 as "Gen 2" next to LR2021 as "Gen 4" implies a direct lineage that doesn't exist at the SPI protocol level.

## Migration

### Files to deprecate (SX1280 raw SPI):
- `firmware/rp2040/src/flrc_range_rx_auto.cpp` — uses SX1280 GET_PACKET_STATUS
- `firmware/rp2040/src/flrc_range_rx_gps.cpp` — uses SX1280 GET_PACKET_STATUS
- `firmware/rp2040/src/flrc_throughput_tx.cpp` — SX1280 SPI commands
- `firmware/rp2040/src/flrc_throughput_rx.cpp` — SX1280 SPI commands
- `firmware/rp2040/src/flrc_raw_tx_pipe.cpp` — SX1280 SPI commands

### Replacement (RadioLib LR2021 driver):
- `firmware/rp2040-flrc-max/` — uses RadioLib's `LR2021` class with `beginFLRC()`
- This is the ONLY correct firmware path forward

## Consequences
- All future firmware uses RadioLib LR2021 driver exclusively
- Raw SPI register access only via LR11xx command set (SWDR001 reference)
- The SX1280-tainted files remain in git history but are deprecated in place
- New contributors and LLM sessions see the ban immediately in AGENTS.md and this ADR
